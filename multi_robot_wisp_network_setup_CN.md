# Unitree GO2 多机器人 WISP/NAT 组网教程
近期在开展多机器人导航相关真机实验时，主包发现 Unitree GO2 本体并不自带可直接用于实验组网的无线网卡。如果完全依赖有线连接，不仅现场布线复杂，也不利于后续 demo 展示和多机器人协同实验。目前公开资料中较少有面向多台 GO2 真机组网的完整教程。从部分相关工作和实验系统配图来看，常见方案主要包括无线网卡、机载路由器和机载主机三种。

对于无线网卡方案，可参考教程：https://www.youtube.com/watch?v=Dsntvdr2cho

对于机载主机方案，思路相对直接：为每台机器人外接一台具备无线网卡的小型主机，使其与服务器处于同一局域网内即可。

本项目主要围绕第二种方案，即通过机载路由器实现多台 GO2 接入同一局域网，并进一步实现服务器对多台机器人进行 SSH 连接与基本通信测试。该方案尽量不修改 GO2 本体的永久网络配置，而是通过临时 IP 的方式完成调试，整体风险较低，便于复现和扩展。本项目不局限于特定品牌路由器和机器人。对于其他支持 WISP 功能的路由器，或其他单机器人、多机器人平台，可根据实际网络环境调整对应 IP 地址和设备名称。
由于个人经验有限，文中若有不妥之处，欢迎交流与批评指正。



本文档介绍一种适合多台 Unitree GO2 同时接入同一服务器的网络配置方式。每台 GO2 连接一台独立的机载路由器，机载路由器通过 **WISP/NAT** 模式接入上级 `BASE` 网络。服务器只访问每台机载路由器在 `BASE` 网络中的地址和端口映射，不直接进入每台 GO2 的私有 LAN。

核心原则：

```text
1. 每台 GO2 本体保持默认地址：192.168.123.18/24
2. 每台 GO2 独占一个机载路由器私有 LAN
3. 每台机载路由器使用 WISP/NAT 接入 BASE 网络
4. 服务器通过端口映射访问 SSH、ZMQ 等显式 TCP 服务
5. 不把多台 GO2 的私有 LAN 合并成同一个二层网络
```

这样可以让多台 GO2 同时使用相同的本体默认地址，同时避免不同机器人之间的底层通信互相可见。

---

## 1. 适用场景

本教程适用于以下场景：

```text
1. 一台服务器需要同时连接两台或更多 Unitree GO2
2. 每台 GO2 都有一台独立的机载路由器
3. 上级网络中有一个统一的 BASE 主路由器或 Wi-Fi 热点
4. 服务器与机载路由器都连接到 BASE 网络
5. 服务器通过 SSH、ZMQ TCP 等明确端口访问每台 GO2
```

本教程默认使用两台 GO2 作为示例。如果需要扩展到更多机器人，只需要为每台机器人增加一台机载路由器，并分配不同的 `BASE` 侧地址和外部端口。

---

## 2. 网络架构总览

### 2.1 推荐拓扑

示例网络拓扑如下。实际使用时，请以 `BASE` 主路由器后台显示的 IP 为准。

```text
BASE 主路由器
IP：192.168.0.1
网段：192.168.0.0/24

├── 服务器
│   └── 192.168.0.132
│
├── GO2-1 机载路由器
│   ├── 工作模式：WISP/NAT
│   ├── 上级 Wi-Fi：BASE 或 BASE 5G
│   ├── BASE 侧 IP：192.168.0.135
│   ├── LAN IP：192.168.123.1/24
│   └── GO2-1 本体 eth0：192.168.123.18/24
│
└── GO2-2 机载路由器
    ├── 工作模式：WISP/NAT
    ├── 上级 Wi-Fi：BASE 或 BASE 5G
    ├── BASE 侧 IP：192.168.0.105
    ├── LAN IP：192.168.123.1/24
    └── GO2-2 本体 eth0：192.168.123.18/24
```

### 2.2 地址规划

| 设备 | 用途 | 示例 |
|---|---|---|
| BASE 主路由器 | 上级网关 | `192.168.0.1` |
| 服务器 | 控制端 | `192.168.0.132` |
| GO2-1 机载路由器 | GO2-1 的 BASE 侧入口 | `192.168.0.135` |
| GO2-2 机载路由器 | GO2-2 的 BASE 侧入口 | `192.168.0.105` |
| GO2-1 路由器 LAN | GO2-1 私有网关 | `192.168.123.1` |
| GO2-2 路由器 LAN | GO2-2 私有网关 | `192.168.123.1` |
| GO2-1 本体 | GO2-1 私有 LAN 内地址 | `192.168.123.18` |
| GO2-2 本体 | GO2-2 私有 LAN 内地址 | `192.168.123.18` |

两台机载路由器的 LAN 侧地址可以相同，因为它们分别位于不同的 NAT 私有网络内。服务器侧访问的是不同的 `BASE` 侧 IP 和端口，不会直接访问 `192.168.123.18`。

### 2.3 端口规划

| 机器人 | 机载路由器 BASE 侧 IP | 服务 | 外部端口 | 内部地址 | 内部端口 |
|---|---:|---|---:|---:|---:|
| GO2-1 | `192.168.0.135` | SSH | `2211` | `192.168.123.18` | `22` |
| GO2-1 | `192.168.0.135` | TCP | `5557` | `192.168.123.18` | `5557` |
| GO2-2 | `192.168.0.105` | SSH | `2212` | `192.168.123.18` | `22` |
| GO2-2 | `192.168.0.105` | TCP | `5557` | `192.168.123.18` | `5557` |

服务器访问示例：

```bash
ssh -p 2211 unitree@192.168.0.135
ssh -p 2212 unitree@192.168.0.105
```

ZMQ TCP 地址示例：

```text
GO2-1：tcp://192.168.0.135:5557
GO2-2：tcp://192.168.0.105:5557
```

## 3. 准备工作



### 3.1 网络前提

配置前请确认：

```text
1. BASE 主路由器可以正常提供 Wi-Fi 和 DHCP
2. 服务器已经连接到 BASE 网络
3. 两台机载路由器可以连接到 BASE Wi-Fi
4. 每台 GO2 可以通过 eth0 连接自己的机载路由器 LAN 口
5. 可以进入每台机载路由器的管理后台
```

### 3.2 示例地址替换规则

本文使用以下示例地址：

```text
服务器：192.168.0.132
GO2-1 机载路由器 BASE 侧 IP：192.168.0.135
GO2-2 机载路由器 BASE 侧 IP：192.168.0.105
```

实际配置时，请在 `BASE` 主路由器后台查看每台机载路由器拿到的真实 IP，并将后续命令中的示例地址替换为现场地址。

## 4. 物理连接

每台 GO2 只连接自己的机载路由器 LAN 口：

```text
GO2-1 eth0  <-网线->  GO2-1 机载路由器 LAN 口
GO2-2 eth0  <-网线->  GO2-2 机载路由器 LAN 口
```

机载路由器通过 Wi-Fi 连接上级 `BASE` 网络：

```text
GO2-1 机载路由器  <-Wi-Fi WISP/NAT->  BASE
GO2-2 机载路由器  <-Wi-Fi WISP/NAT->  BASE
服务器            <-Wi-Fi 或有线->     BASE
```

接线检查：

```text
1. GO2 网线接机载路由器 LAN 口
2. 服务器连接 BASE 网络
3. GO2-1 和 GO2-2 分别连接自己的机载路由器
4. 两台机载路由器都通过 WISP/NAT 接入 BASE
5. 两台 GO2 本体都保持 192.168.123.18/24
```

## 5. 机载路由器配置

以下步骤需要分别在 `GO2-1` 和 `GO2-2` 的机载路由器后台中完成。不同品牌路由器的菜单名称可能略有差异，请以实际界面为准。

### 5.1 设置 WISP/NAT 工作模式

进入机载路由器后台，找到工作模式设置页面，例如：

```text
更多功能 -> 工作模式
```

选择：

```text
热点信号放大模式 (WISP)
```

或其他等价名称：

```text
WISP
WISP 路由
无线 ISP
无线中继路由模式
NAT 中继模式
```

上级 Wi-Fi 选择：

```text
BASE 或 BASE 5G
```

保存配置后，等待路由器重启并重新接入 `BASE` 网络。

### 5.2 设置 LAN 地址

进入 LAN 设置页面，例如：

```text
更多功能 -> 网络设置 -> 局域网设置
```

将每台机载路由器的 LAN 侧地址设置为：

```text
局域网 IP 地址：192.168.123.1
子网掩码：255.255.255.0
```

保存后，机载路由器的 LAN 侧后台地址通常会变为：

```text
http://192.168.123.1
```

由于每台机载路由器的 LAN 都是独立私有网络，`GO2-1` 和 `GO2-2` 的 LAN 侧都可以使用 `192.168.123.1/24`。

### 5.3 配置 GO2-1 端口映射

进入端口映射页面，例如：

```text
更多功能 -> 高级设置 -> 端口映射
```

添加 SSH 端口映射：

```text
选择设备：手动
内网 IP 地址：192.168.123.18
内网端口：22
外网端口：2211
协议：TCP
```

添加 ZMQ TCP 端口映射：

```text
选择设备：手动
内网 IP 地址：192.168.123.18
内网端口：5557
外网端口：5557
协议：TCP
```

### 5.4 配置 GO2-2 端口映射

添加 SSH 端口映射：

```text
选择设备：手动
内网 IP 地址：192.168.123.18
内网端口：22
外网端口：2212
协议：TCP
```

添加 ZMQ TCP 端口映射：

```text
选择设备：手动
内网 IP 地址：192.168.123.18
内网端口：5557
外网端口：5557
协议：TCP
```

如果路由器界面只提供 `TCP&UDP`，可以先选择 `TCP&UDP`。SSH 与 ZMQ TCP 实际使用的是 TCP。

### 5.5 确认机载路由器的 BASE 侧 IP

服务器切回 `BASE` 网络后，进入 `BASE` 主路由器后台：

```text
设备管理 / 在线设备 / DHCP 客户端列表
```

找到两台机载路由器，并记录它们的 `BASE` 侧 IP。示例：

```text
GO2-1 机载路由器：192.168.0.135
GO2-2 机载路由器：192.168.0.105
```

如果现场 IP 与示例不同，后续命令以现场 IP 为准。

### 5.6 防火墙、DMZ 与 UPnP 建议

建议保持以下配置：

```text
1. DMZ 主机：关闭
2. UPnP：关闭或不使用
3. 端口映射：只开放需要的 TCP 端口
4. 静态路由：不需要在 BASE 中添加 192.168.123.0/24
```

部分路由器开启 `防 WAN 口 Ping` 后，服务器可能无法 `ping` 通机载路由器。这不一定代表端口映射失败，应以 `nc` 和 `ssh` 测试结果为准。

## 6. GO2 本体网络配置

本节需要先连接到对应机载路由器的 Wi-Fi，进入每台 GO2 的私有 LAN 后执行。

### 6.1 直连 GO2 本体

连接 `GO2-1` 机载路由器自己的 Wi-Fi 后，测试 GO2 本体：

```bash
ping -c 3 192.168.123.18
nc -vz -w 3 192.168.123.18 22
ssh unitree@192.168.123.18
```

连接 `GO2-2` 机载路由器自己的 Wi-Fi 后，执行相同检查：

```bash
ping -c 3 192.168.123.18
nc -vz -w 3 192.168.123.18 22
ssh unitree@192.168.123.18
```

进入 GO2 后，确认 `eth0` 地址：

```bash
ip -4 -br addr show eth0
```

期望看到：

```text
eth0 UP 192.168.123.18/24
```

### 6.2 添加回程路由

每台 GO2 需要知道如何把返回服务器的数据包交给自己的机载路由器。假设服务器位于 `192.168.0.0/24` 网段，机载路由器 LAN 地址为 `192.168.123.1`，在每台 GO2 本体上执行：

```bash
sudo ip route replace 192.168.0.0/24 via 192.168.123.1 dev eth0
```

如果服务器或 `BASE` 网络使用其他网段，请将 `192.168.0.0/24` 替换为现场网段。

### 6.3 验证回程路径

假设服务器地址是 `192.168.0.132`，在 GO2 本体上执行：

```bash
ip route get 192.168.0.132
```

期望看到类似输出：

```text
192.168.0.132 via 192.168.123.1 dev eth0 src 192.168.123.18
```

这表示 GO2 发往服务器的数据会经由机载路由器返回 `BASE` 网络。

### 6.4 让回程路由开机生效

如果现场系统重启后会丢失临时路由，需要将回程路由写入开机脚本或网络配置。最小可用方式是在 GO2 上创建一个启动时执行的脚本，内容包含：

```bash
sudo ip route replace 192.168.0.0/24 via 192.168.123.1 dev eth0
```

写入持久化配置前，请先用 `ip route get` 确认路由正确，再根据现场系统的启动方式固化。

## 7. 服务器侧访问验证

以下命令都在服务器上执行。服务器需要连接到 `BASE` 网络。

### 7.1 确认服务器位于 BASE 网络

检查当前 Wi-Fi：

```bash
nmcli -t -f ACTIVE,SSID dev wifi | grep '^yes'
```

检查到 `BASE` 网关的路由：

```bash
ip route get 192.168.0.1
```

测试 `BASE` 网关连通性：

```bash
ping -c 5 192.168.0.1
```

### 7.2 测试 GO2-1 SSH

假设 `GO2-1` 机载路由器的 `BASE` 侧 IP 是 `192.168.0.135`：

```bash
nc -vz -w 3 192.168.0.135 2211
ssh-keygen -f ~/.ssh/known_hosts -R "[192.168.0.135]:2211"
ssh -p 2211 unitree@192.168.0.135
```

连接成功后，说明服务器可以通过端口映射进入 `GO2-1` 本体。

### 7.3 测试 GO2-2 SSH

假设 `GO2-2` 机载路由器的 `BASE` 侧 IP 是 `192.168.0.105`：

```bash
nc -vz -w 3 192.168.0.105 2212
ssh-keygen -f ~/.ssh/known_hosts -R "[192.168.0.105]:2212"
ssh -p 2212 unitree@192.168.0.105
```

连接成功后，说明服务器可以通过端口映射进入 `GO2-2` 本体。

### 7.4 远程检查 GO2 本体地址

在服务器上分别检查两台 GO2 的 `eth0`：

```bash
ssh -p 2211 unitree@192.168.0.135 'ip -4 -br addr show eth0'
ssh -p 2212 unitree@192.168.0.105 'ip -4 -br addr show eth0'
```

期望两台都显示：

```text
eth0 UP 192.168.123.18/24
```

### 7.5 测试 ZMQ TCP 端口

如果 GO2 本体上已经启动监听 `5557` 的 ZMQ TCP 服务，可以在服务器上测试端口：

```bash
nc -vz -w 3 192.168.0.135 5557
nc -vz -w 3 192.168.0.105 5557
```

期望看到连接成功。此时服务器访问的是两台机载路由器的 `BASE` 侧 IP，但请求会被分别转发到各自私有 LAN 内的 `192.168.123.18:5557`。

### 7.6 SSH 端口通但登录卡住时的检查

如果 `nc` 显示端口可达，但 `ssh` 登录卡住，例如：

```bash
nc -vz -w 3 192.168.0.135 2211
ssh -p 2211 unitree@192.168.0.135
```

先检查 SSH banner：

```bash
timeout 10 nc -v 192.168.0.135 2211
```

正常情况下应看到类似：

```text
SSH-2.0-OpenSSH_...
```

如果看不到 banner，优先回到对应 GO2 本体确认回程路由：

```bash
sudo ip route replace 192.168.0.0/24 via 192.168.123.1 dev eth0
ip route get 192.168.0.132
```

如果能看到 banner 但 SSH 仍然卡住，可以使用详细日志排查：

```bash
ssh -vvv \
  -o ConnectTimeout=10 \
  -o GSSAPIAuthentication=no \
  -o IPQoS=none \
  -p 2211 unitree@192.168.0.135
```

`GO2-2` 的检查方式相同，只需要将 IP 和端口替换为现场配置，例如：

```bash
timeout 10 nc -v 192.168.0.105 2212
ssh -vvv \
  -o ConnectTimeout=10 \
  -o GSSAPIAuthentication=no \
  -o IPQoS=none \
  -p 2212 unitree@192.168.0.105
```

完成以上验证后，服务器即可通过 `BASE` 网络分别访问两台 GO2。每台 GO2 仍保留在自己的 `192.168.123.0/24` 私有 LAN 内，多台机器人之间不会因为相同的本体默认地址产生冲突。
